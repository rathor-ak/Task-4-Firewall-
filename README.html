Cyber Security Internship - Task 4

Firewall Configuration using UFW (Linux)

Performed Steps

  1. Checked UFW status
  2. Enabled the firewall
  3. Listed current rules
  4. Blocked port 23 (Telnet)
  5. Tested the Telnet block rule
  6. Ensured SSH port 22 was allowed
  7. Removed the test block rule

Command Outputs

sudo ufw status
=> Status: inactive

sudo ufw enable
=> Firewall is active and enabled on system startup

sudo ufw status numbered
=> Status: active
   To                         Action      From
   --                         ------      ----
   22                         ALLOW       Anywhere
   22 (v6)                    ALLOW       Anywhere (v6)

sudo ufw deny 23
=> Rule added (v6)

telnet localhost 23
=> Connection refused

sudo ufw delete deny 23
=> Rule deleted (v6)

Summary

We successfully configured a basic firewall using UFW, tested blocking of Telnet, preserved SSH, and restored original rules.

Interview Questions Answered

  • Firewall: Monitors and controls traffic.
  • Stateful vs Stateless: Tracks state vs per-packet check.
  • Inbound/Outbound: Control incoming vs outgoing traffic.
  • UFW: Simplifies iptables configuration.
  • Why block 23: Telnet is insecure.
  • Mistakes: Blocking critical ports, not enabling firewall.
  • Security: Blocks malicious traffic, enforces policy.
  • NAT: Translates private to public IP.
